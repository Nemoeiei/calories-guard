Security Posture

Transport · AuthN · AuthZ · Rate limiting · Uploads · Secrets · Monitoring
🔒 Transport

Railway fronts TLS 1.2+; backend sees X-Forwarded-Proto: https. All traffic encrypted in transit.

🪪 AuthN

Supabase issues JWT (HS256 · SUPABASE_JWT_SECRET). Backend verifies signature + exp, maps sub → cleangoal.users.user_id.

🛡 AuthZ

Route-level dep get_current_user() / get_current_admin() (role_id == 1). RLS on user-owned tables as second line of defence.

⏱ Rate Limiting

slowapi — /login 5/15min/IP, /chat/* 10/hour/user, /upload-image 10/min/user.

📎 Upload Validation

5 MB cap, MIME whitelist image/jpeg|png|webp. Enforced at the FastAPI layer before forwarding to Supabase Storage.

🔑 Secrets

Never in repo — Railway env vars + flutter_secure_storage on device. .env.example documents keys only.

📊 Monitoring

Sentry captures exceptions in both backend and Flutter client. UptimeRobot pings /health every 5 minutes.

Data Flow Summary

Source → Sink, transport mechanism, and key notes for each flow
Source Sink Transport Notes
Flutter App FastAPI HTTPS REST + Bearer JWT Central api_client.dart handles 401 → logout
FastAPI Supabase Postgres psycopg2 over pgbouncer (6543) Pooled, SSL required
FastAPI Gemini HTTPS REST 30s timeout + 1 retry
FastAPI Supabase Storage REST (service role) Signed URLs returned to client
Flutter Supabase Auth HTTPS (GoTrue) Email/OTP, Google OAuth
Backend / Flutter Sentry HTTPS Errors + 10% traces
UptimeRobot Railway HTTPS GET /health 5-minute interval
